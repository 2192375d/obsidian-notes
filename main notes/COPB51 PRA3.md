# types of interview questions
- __introductory__
	_First/last_ question in the interview, a good chance to leave a strong first/last impression
	Solved using _PPF_

- __behavioural__
	Behavioral questions that focuses on the _past_ experiences.
	Solved using _STARR_

- __motivation/trait/open__
	vague questions that provide a chance to highlight your _personality, values and motivations_.
	Solved using _PEC_ and _STARR_

- __accomplishment-oriented__
	questions focus on your achievements and successes, emphasizing measurable results.
	Solved using _PEC_

- __informational/skills-based__
	questions checking if a candidate has a _specific skill_
	Solved using _PEC_

- __situational/case__
	behavioral questions but instead it focuses on the _future_
	Solved using _STARR_ and _DAE_

- __negative/trap__
	Questions aim to reveal a candidate's weakness, often done by probing to failures.
	(to deal with this, don't say you are perfect, employers want to know if you are self-aware)
	Solved using _Sandwich method_

- __technical (conceptual)__
	Tech-heavy questions.
	Solved using _DAE_ and _ACE_

# interview questions respond strategy
- __PEC__
	Point + Example + Conclusion
	_Point_: State the strength, skill
	_Example_: choose examples that align with job requirement, be specific
	_conclusion_: connect example back to employer's needs

- __STARR__
	Situation + Task + Action +Result + Relate

- __PPF__
	Present + Past + Future
	For the future part, relate it to present and past

- __Sandwich__
	Start Positive + Weakness + End Positive
	_Start Positive_: Highlight a strength or positive trait
	_Weakness_: Address an area of improvement honestly, but <u>constructively</u>
	_End Positive_: Conclude by emphasizing what works are done to imrpove the weakness. 

- __DAE__
	Define + Answer + Example
	_Define_: Explain what the question from the interviewer is asking

- __ACE__
	Ask + Communicate + Evaluate
	_Ask_: Ask questions to help you create variables and functions
	_Communicate_: Explain the thought process for the problem
	_Evaluate_: Use test cases to identify base case and mistakes

# interview formats
- __individual interview__
	Try to see it as a conversation
- __group interview__
	A group of people competing for a job. It is important to think about how to play _a good role in group_
- __panel interview__
	A bunch of people interview you. Try to make eye contact with _all_ of the interviewers.

# interview format
Only _4%_ of the interviews are in person.
For online interviews, make sure you look professional enough on the platforms. Make sure you are dressed well.

# what to do if I don't know the answer of a question?
You can ask for a break of 1 minute to think about it, or worse case scenario request to respond at the end of the interview.
